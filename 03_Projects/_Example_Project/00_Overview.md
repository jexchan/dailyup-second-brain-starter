# Overview

> 这是一个**示例项目**，展示 `03_Projects/` 下一个完整项目是什么样子。
> 新建真实项目时，请复制 `.templates/` 里的骨架并删掉本示例。

## 项目名称
Example Project — 一个用来演示目录结构的样板项目

## 项目定义
> 一句话说明这个项目是什么、要解决什么问题

帮助新用户在 30 分钟内看懂：一个项目应该包含哪些文件、文件之间怎么流转。

## 项目背景
> 为什么要做这个项目？当前状况如何？

- 许多新用户拿到模板后不知道每个文件该写什么
- 纯文字说明不够直观，需要一个完整的实例做参考
- 本示例刻意保持简洁，突出流程而非内容

## 当前阶段
active

## 成功标准
> 如何判断这个项目完成了、做好了？

- 新用户看完 8 个文件后能独立新建自己的项目
- 项目下的任务可以在 `03_Backlog` → `04_Next` → `05_Doing` → `06_Testing` → `07_Done` 之间流转
- 项目的目标、决策、下一步清晰可查

## 关键约束
> 时间、资源、范围等方面的限制

- 仅用于演示，不承载任何真实工作
