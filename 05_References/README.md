# References 剪藏文章管理

## 目录结构

### 01_Inbox
新剪藏文章的入口，所有未分类、未阅读的文章先放这里。

### 02_AI_&_Tech
AI 和技术相关文章

### 03_Education
教育、学习科学相关文章

### 04_Product
产品、设计、商业模式相关文章

### 05_Writing
写作、内容创作相关文章

### 06_Research
研究论文、深度报告、学术资料

### 07_Archive
已消化/过期的原始文章，保留备查

## 工作流程

```
网上剪藏 → 01_Inbox/
           ↓
       阅读/分类
           ↓
       移至对应主题目录
           ↓
       提炼知识卡片 → 04_Knowledge/00_Cards/
           ↓
       原文移至 07_Archive/ 或删除
```

## 原则

- **定期清理 Inbox**，避免堆积
- **只保留有价值的原文**，其他及时删除
- **优先转化为知识卡片**，而不是囤积原文
- **与项目强相关**的直接移到对应项目目录

## 文件命名建议

```
YYYY-MM-DD - 原标题.md
```

例如：`2026-04-23 - 如何用 AI 辅助课程设计.md`
