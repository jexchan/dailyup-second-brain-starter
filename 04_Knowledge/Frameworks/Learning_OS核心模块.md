# Learning OS 核心模块

## 一句话定义
Learning OS 是一套面向 AI 时代的学习操作系统，帮助学习者建立从输入到输出的完整学习闭环。

## 核心框架

```
                    ┌─────────────────┐
                    │    AI Support   │
                    │  结构化·启发    │
                    │  反馈·执行      │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        ▼                    ▼                    ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│    Input      │   │   Organize    │   │  Retrieval    │
│  输入与捕获    │   │  结构化整理   │   │   提取练习    │
└───────┬───────┘   └───────┬───────┘   └───────┬───────┘
        │                   │                   │
        └───────────────────┼───────────────────┘
                            ▼
                   ┌───────────────┐
                   │  Application  │
                   │  迁移与应用   │
                   └───────┬───────┘
                           ▼
                   ┌───────────────┐
                   │    Output     │
                   │   输出与创造  │
                   └───────────────┘
```

## 各模块说明

### Input — 输入与捕获
- **目标**：高效捕获有价值的信息
- **关键**：有选择的输入，而非被动消费
- **AI 助力**：快速总结、提取要点

### Organize — 结构化整理
- **目标**：建立知识之间的连接
- **关键**：结构优于收藏，关系优于孤立
- **AI 助力**：自动分类、发现关联

### Retrieval — 提取练习
- **目标**：让知识可被调用
- **关键**：主动提取 > 被动重读
- **AI 助力**：生成练习题、自测工具

### Application — 迁移与应用
- **目标**：把知识用到新场景
- **关键**：情境变化、问题迁移
- **AI 助力**：场景设计、变式练习

### Output — 输出与创造
- **目标**：用知识创造新价值
- **关键**：输出是学习的一部分
- **AI 助力**：辅助表达、生成草稿

## 核心思想
- 学习不是信息消费，而是认知结构的变化
- 提取比重读更重要
- 输出是学习的一部分，而非附属品
- AI 应该辅助而非替代思考
