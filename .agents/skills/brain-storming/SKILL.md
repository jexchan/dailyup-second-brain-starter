# 头脑风暴技能 (Brainstorming)

这个技能帮助你围绕任何主题进行深度的发散性思维探索，然后由你来收敛和沉淀。

## 核心原则

1. **发散先行**：先不评价、不筛选，尽可能多地产生想法
2. **多维探索**：从不同角度、不同层次切入主题
3. **保持开放**：欢迎离奇的联想、反向的思考、跨域的类比
4. **你来收敛**：发散完成后，由你来审视、筛选、整合
5. **沉淀入库**：最终结果写回知识库，成为长期资产

---

## 第一步：发散探索

当用户给出一个主题时，从以下维度进行发散（不必每次都用全部，选择最相关的维度）：

### 1. 联想发散
- **向上抽象**：这个主题属于什么更大的范畴？本质是什么？
- **向下具体**：可以分解成哪些子概念？有哪些具体实例？
- **横向关联**：相似的东西是什么？相关的东西是什么？

### 2. 类比思维
- **自然界的类比**：自然界中有什么类似的现象/结构？
- **跨领域类比**：其他领域（科学、艺术、历史、商业）中有什么类似的概念？
- **日常类比**：日常生活中有什么可以类比的场景？

### 3. 反向思考
- **否定命题**：如果这个主题的反面是什么？会有什么启发？
- **颠覆假设**：这个主题背后的默认假设是什么？如果推翻会怎样？
- **极端情况**：推向极端会怎样？完全消失会怎样？

### 4. 延伸推演
- **时间维度**：过去是怎样的？未来可能怎样演化？
- **空间维度**：不同环境下会有什么变化？
- **因果链条**：导致它的原因是什么？它会引发什么后果？

### 5. 跨领域连接
- **跨学科视角**：从物理学、生物学、心理学、哲学等不同学科怎么看？
- **跨文化视角**：不同文化背景下这个主题有什么不同含义？
- **跨时代视角**：古人怎么想？现代人怎么想？未来人可能怎么看？

### 6. 质疑与批判
- **问题风暴**：关于这个主题，有哪些值得问的问题？
- **质疑前提**：大家默认的前提真的成立吗？
- **批判性思考**：可能的反对意见是什么？局限性是什么？

### 7. 创意组合
- **元素重组**：相关元素可以怎么重新组合？
- **混搭融合**：这个主题可以和什么看似不相关的东西结合？
- **变异改造**：做一点小改动会产生什么新意？

## 输出格式

使用以下结构化格式输出发散结果：

```markdown
# 头脑风暴：[主题]

## 发散探索

### 1. 联想发散
- 向上抽象：...
- 向下具体：...
- 横向关联：...

### 2. 类比思维
- 自然界类比：...
- 跨领域类比：...

### 3. 反向思考
- 否定命题：...
- 颠覆假设：...

### 4. 延伸推演
- 时间维度：...
- 因果链条：...

### 5. 跨领域连接
- ...

### 6. 质疑与批判
- ...

### 7. 创意组合
- ...
```

**注意**：
- 每个维度给出 3-5 个具体条目
- 保持条目简洁有力，一条一个点
- 使用项目符号 `-` 列出
- 鼓励具体的、有启发性的内容，避免空泛

## 第二步：等待用户收敛

发散完成后，**明确停止**，等待用户的反馈：

```
---
*发散完成。以上是从多个维度对「[主题]」的探索。*

*接下来请你审视这些想法：*
- *哪些点对你有启发？*
- *哪些值得进一步深挖？*
- *你想保留什么、丢弃什么？*

*告诉我你的想法，我来帮你整合并沉淀到知识库。*
```

**重要**：不要自动进入收敛阶段，等待用户的明确指示。

---

## 第三步：整合与沉淀

当用户给出收敛指示后，按以下步骤处理：

### 1. 理解用户的收敛意图
- 用户选出了哪些有价值的点？
- 用户有什么新的补充或调整？
- 用户想强调什么核心洞见？

### 2. 整合输出
基于用户的反馈，输出一份整合后的文档，包含：
- **核心洞见**：最重要的发现
- **关键联想**：有价值的关联
- **待探索问题**：未解决的问题或值得进一步思考的方向
- **行动/后续**（如用户提到）：下一步做什么

### 3. 沉淀到知识库
确认保存位置后，将整合内容写入 `04_Knowledge/` 目录：
- 如果已有相关主题文件，追加或更新
- 如果没有，创建新文件
- 遵循 Vault 的命名规范和写作规则
- 使用清晰的标题结构

**写入前确认**：
```
我将把这份内容保存到 `04_Knowledge/[主题].md`。
如果这个位置不对，请告诉我应该放哪里。
```

## 特殊情况处理

### 用户一开始就指定了发散维度
如果用户说"从XX角度想想"或"用XX框架分析"，则聚焦于用户指定的维度，不必面面俱到。

### 用户的主题非常抽象
对于抽象主题（如"自由""意义""时间"）：
- 先用具体例子落地
- 再进行发散
- 发散后回到抽象层面总结

### 用户的主题很具体狭窄
对于具体主题（如"怎么养多肉"）：
- 先向上抽象到更大范畴（如"植物养护""照料生命"）
- 再进行发散
- 发散后回到具体场景

### 用户想要"更多"
如果用户说"再想想""还有吗""不够多"：
- 深入某个维度，给出更细致的条目
- 或者启用之前没用过的维度
- 避免重复已有的内容
